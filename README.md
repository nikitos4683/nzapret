**nzapret** — автономный и легковесный модуль для Android-устройств с root-доступом. Он использует локальный движок `nfqws` и правила `iptables`/`ip6tables` для обхода DPI.

Проект опирается на наработки оригинального [bol-van/zapret](https://github.com/bol-van/zapret). При этом бинарники, пейлоады и списки маршрутизации (lists) взяты именно оттуда, но весь код обвязки написан с нуля под Android — с красивым интерфейсом и без лишнего мусора в системе.

---

## ✨ Ключевые возможности

- **Встроенный WebUI для KernelSU:** Детально проработанный и красивый графический интерфейс прямо в вашем менеджере модулей.
- **Мощный CLI:** Полный контроль над поведением модуля через терминал.
- **Легковесная стратегия "из коробки":** Базовый профиль рассчитан на мобильные сети и смартфоны и не требует дополнительной настройки для базового сценария.
- **Умный роутинг:** Незаметный перехват трафика в цепочках `OUTPUT` и `FORWARD` для IPv4 и IPv6.
- **Поддержка профилей:** Легкое переключение между разными конфигурациями обхода "на лету".
- **Персональный `list-user`:** Собственный список доменов пользователя с редактированием из WebUI и CLI.
- **Локальная автономность:** Служба не выходит в сеть при загрузке устройства и использует только локальные данные.
- **Горячее обновление списков:** Скачивание и обновление доменных листов маршрутизации вызовом одной команды.
- **Совместимость с VPN:** Модуль рассчитан на совместную работу с большинством VPN-клиентов. *(Если ваш VPN использует экзотические сетевые интерфейсы и возникают конфликты — смело открывайте Issue, мы постараемся это исправить!)*

## 🏗 Архитектура и структура

Модуль состоит из нескольких компонентов, которые работают в единой слаженной связке:

### ⚙️ Ядро: `service.sh` и CLI

- **`service.sh`** — главный системный скрипт модуля. Стартует вместе с системой, поднимает выбранный профиль обхода, пересобирает правила фаервола под текущую конфигурацию и запускает процесс `nfqws`.
- **`system/bin/nzapret`** — основной мозг управления (CLI). Предоставляет команды для запуска, остановки, перезапуска рантайма и обновления листов (hostlists). Также именно CLI генерирует телеметрию в JSON-формате для работы WebUI.
- **`uninstall.sh` / `action.sh`** — утилиты для корректной очистки правил `iptables`, остановки запущенных процессов и быстрой работы кнопки Вкл/Выкл в интерфейсе.

### 🎨 WebUI (Интерфейс)

Модуль содержит собственный Web-интерфейс (`webroot/`), который интегрируется в KernelSU.
- Отображает подробный статус устройства (версия, PID, количество правил и доменов).
- Позволяет управлять модулем (Start / Stop / Restart / Выбор профиля).
- **Встроенный визуальный редактор** для управления персональным списком доменов (`list-user`).
- Содержит вкладки с системными логами службы и логами диагностики в реальном времени.
- Имеет встроенный режим диагностики проблем, помогающий выявить причины неполадок на конкретном устройстве.
- Вся связь между UI и модулем работает через вызовы к CLI (`nzapret status --json`).

### 📂 Данные и Конфигурация

- **`profiles/`** — конфигурации с аргументами `nfqws` для разных сценариев обхода блокировок.
- **`lists/`** — текстовые списки доменов для выборочного роутинга трафика.
  Встроенные листы обновляются отдельно, а персональный `list-user` хранится как пользовательский runtime-файл и не должен теряться при обновлении модуля.
- **`payloads/`** — бинарные фейковые пакеты (TLS/QUIC) для обмана оборудования провайдеров.

## 🚀 Быстрый старт

### 1. Установка
1. Скачайте готовый `ZIP-архив` со страницы релизов (Releases) или соберите модуль из исходников самостоятельно.
2. Прошейте стандартным способом через **Magisk** или **KernelSU**.
3. Перезагрузите устройство.
4. Откройте вкладку модулей в KernelSU для доступа к **WebUI**. *(Если вы используете Magisk, установите приложение [KsuWebUI](https://github.com/KOWX712/KsuWebUIStandalone) для работы с интерфейсом, либо управляйте модулем по старинке через терминал).*

### 2. Использование из терминала

Для управления через CLI потребуется терминал (например, Termux) и root-права (команда `su`).
*Примечание: Если команда `nzapret` не найдена в системном пути, используйте полный путь: `/data/adb/modules/nzapret/system/bin/nzapret`*

**Жизненный цикл:**
```sh
nzapret start    # Инициализировать правила и запустить nfqws
nzapret stop     # Остановить процесс и очистить цепочки фаервола
nzapret restart  # Полный перезапуск с применением новых настроек
```

**Мониторинг:**
```sh
nzapret status         # Посмотреть текущее состояние и статус службы
nzapret status --json  # Вывод статуса машинной обработкой (для WebUI)
nzapret log            # Посмотреть последние записи системного лога
nzapret diagnose       # Собрать и показать диагностическую информацию
```

**Листы и профили:**
```sh
nzapret update         # Скачать и обновить списки обхода (hostlists)
nzapret profile list   # Показать доступные профили конфигурации
nzapret profile set <name> # Переключить активный профиль
nzapret list-user status    # Статус персонального списка и домены
nzapret list-user show      # Показать текущее содержимое
nzapret list-user add example.com youtube.com
nzapret list-user remove example.com
nzapret list-user replace example.com github.com # Перезаписать список
nzapret list-user clear     # Полностью очистить список
```

Если вы используете собственный `profiles/*.conf`, добавьте в нужные секции строку:
```sh
--hostlist=/data/adb/modules/nzapret/lists/list-user.txt
```
Иначе `list-user` будет сохраняться, но не будет применяться активным профилем.

## 🛠 Сборка модуля

Если вы хотите собрать модуль самостоятельно, у вас в системе (Linux/WSL) должны быть установлены базовые утилиты: `bash`, `zip`, `sed`, `mktemp`.

Процесс сборки запускается всего одной командой:
```sh
bash build.sh
```
На выходе в корневой директории появится готовый к прошивке архив `nzapret-vX.Y.Z.zip`. Собирать можно прямо на устройстве (например, в Termux) или на ПК.
